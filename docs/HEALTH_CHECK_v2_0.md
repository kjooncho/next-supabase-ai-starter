## §2. 강점 — 그대로 유지

### 2.1 산출물 간 인용 무결성 100%
README가 인용한 19개 파일이 모두 존재. 죽은 링크 0건. 사용자가 7월 첫날 README를 첫 페이지로 열고 시작할 때 어느 파일도 사라져 있지 않을 것.

### 2.2 컬러 토큰 일관성
디자인 시스템 시트(Image 8)의 12색 + 텍스트 3색 + 보조 토큰이 prototype.html에서 시스템 외 색상 0건. 단 etymology amber tint 2개(`#854F0B`, `#f5e6d3`)는 의도적 유지.

### 2.3 Eval-first 워크플로우
코딩 전에 Eval 시스템을 셋업한 결정은 AI 기반 프로젝트의 가장 큰 리스크(환각·문화 오역)를 미리 헷지하는 정공법. **외부 사례 부합**: PlayAbly.AI는 사용자 피드백 통합으로 65% → 92%까지 3개월 만에 도달 (Label Your Data, 2026).

### 2.4 시장 정의 정정의 정확성
PRD 5.0의 "1.5만~3만 명" 추정이 외부 통계와 부합:
- 출입국재류관리청 2024년 말: 한국 국적자 **409,238명**
- 특별영주권자: **274,023명** (재일교포, 일본어 학습 불필요)
- 뉴커머: 약 **13만 명** ← 본 앱 타겟
- 그중 N3-N5 학습 의지: **약 1.5만~3만 명** (5-10% 추정)

### 2.5 차별점 재정의의 외부 부합
"AI 페르소나는 차별점 아님" 결정이 외부 트렌드와 부합:
- **Talkpal** (2023년 시작, 2025년 11월 600만 사용자): 이미 Persona Mode 운영 ("Satoshi" 시부야 사업가, "Hana" 교토 정원사)
- **italki**: "AI tools fall short in real conversation and cultural nuance" — 문화 뉘앙스가 LLM의 약점
- **Cultural Nuance Benchmark (Appen 2025)**: 7개 LLM 평균 1.68/3 (56%), 최상위(GPT-5) 2.10/3 (70%). idiom/pun은 더 낮음.

→ "콘텐츠 품질이 진짜 차별점"이라는 결정의 외부 근거 확보.

## §3. 검증 필요 — 7월 첫 주에 확인

### 3.1 Eval Pass Criteria의 Type Accuracy ≥75% 야심성

**문제**: Cultural Nuance Benchmark에서 GPT-5도 70%, Claude Sonnet 3.7은 1.97/3 (66%). Haiku 4.5는 더 낮을 가능성.

**옵션**:
- (a) 첫 실행 결과 70% 미만 → ≥70%로 기준 완화
- (b) 첫 v1은 70% 통과, v2 개선 후 75% 도달의 단계 목표
- (c) Haiku 미달 시 Sonnet으로 변경 ($0.16 → $0.47, 3배)

**권고**: (b) 단계 목표가 가장 합리적. 7월 첫 실행 시 결과에 따라 동적 조정.

### 3.2 캐싱 절감률 49%의 실측 미확인
PRD 5.0이 4.1의 "80-90%" 추정을 정정해 49%로 잡았으나, **실측치가 아닌 재추정**. 실제 측정은 7월 첫 실행 후. 50% 이상이면 다행, 30% 미만이면 비용 재산정 필요.

### 3.3 Meshclass 직접 경쟁자 발견 (신규)

**App Store에서 발견**: Meshclass (https://apps.apple.com/us/app/meshclass-...)
- 일본 기업이 만든 한국인용 일본어 학습 앱
- JLPT N5~N1 6,000+ 문제 + AI 회화 + 손글씨 한자
- 17개 언어 지원
- 한국어 인터페이스 정식 지원

**의미**: PRD 5.0의 "한국 거주자 + 문화 통합이 빈 자리" 차별점이 약화. Meshclass는 한국에 있는 한국인 대상이지만, **JLPT 중심**이라 일본 생활 문화와는 거리가 있음.

**차별점을 더 좁혀야 함**:
- 변경 전: "한국 거주자 + 문화 통합"
- 변경 후: "**일본에 살고 있는** 한국인 + **JLPT에 안 나오는 생활 문화**"

이미 PRD 5.0의 §2 §11에 부분 반영되어 있으나, 명시적 vs Meshclass 비교가 누락. → 패치 1.

### 3.4 베타 리뷰어 1-2명의 부족함
1명이 응답 안 하면 0명이 될 수 있는 single point of failure. **최소 3명** 권장.

### 3.5 베타 리뷰어 섭외 채널 미명시
**일본한국인모임(일한모)** (https://korean.co.jp/) 같은 기존 커뮤니티가 README에 명시되지 않음. 페이스북 그룹, 인스타 한인 커뮤니티 등도 같이 활용해야 응답률 확보 가능. → 패치 2.

## §4. 위험 신호 — 선제 대응 권장

### 4.1 ⚠ 11-12월 콘텐츠 작성 페이스 위기 (가장 큰 위험)

**구조적 문제**:
- Tier 1 8편 + Tier 2 4편 = 12편을 11-12월(약 8주)에 작성
- EP-04 모범 사례 기준 1편 ≈ 220분 (3시간 40분)
- 12편 × 220분 = **44시간 + 검수 반영 30~60분 × 12 = 6~12시간 추가 = 50~56시간**
- 1인 작업이 본업과 병행이라면 평일 평균 1시간 + 주말 4시간 = 주 9시간 → **6주 분량**
- 그러나 10-11월에 코딩(채팅·번역·My Deck·캘린더·생활탭) 동시 진행 → **현실적으로 8편이 한계**

**동기 측면**:
- "포트폴리오 + 검증 무대" 정체성은 수익 인센티브 부재
- 11월 후반에 동기 저하 시점 도래 가능
- 클로즈드 베타 5명도 12월 후반에 시작 → 사용자 반응 받기 전에 동기 고갈

**대응 옵션**:
- **(a) Tier 1 8편 → 5편으로 축소** (お正月, 入園式, 飲み会, 在留카드, 回覧板)
- **(b) 출시 시점을 12월 → 2027년 1월 말로 4-6주 연기**
- **(c) 콘텐츠 작성 외부 위탁** (편당 ¥50,000~¥80,000, 일본 거주 한국인 라이터)

**권고**: (a) + (b) 조합. Tier 1을 5편으로 줄이고, 출시는 1월 말. 출시 후 매월 1-2편씩 추가하는 게 페이스 유지에 유리.

### 4.2 ⚠ 12월 출시 시점의 시장 부적합

**구조적 문제**:
- 12월 후반 = 연말 휴가 시즌 (일본·한국 모두)
- 베타 응답률 평소 50% 대비 30%로 하락 (커뮤니티 활동 둔화)
- 초기 사용자 확보도 1월 첫 주 이후가 유리
- 신규 앱 마케팅: 1월 신년 시즌 vs 12월 연말 → 1월이 압도적 유리

**대응**: 4.1의 권고와 묶어 출시 시점 1월 말로 변경 권장.

### 4.3 ⚠ 변호사 비용 ROI 약함

**문제**:
- 변호사 검토 30~50만원 (또는 ¥30,000~¥50,000)
- 추정 시장 1.5만 명 × 첫해 0.5% 전환 = 75명 × 무료
- 수익 0원 단계에서 50만원 = 영구 손실

**합리적 단계화**:
- **베타 5명 단계** → 동의서 한 장으로 면제 (사용자 5명에게 직접 동의)
- **사용자 100명 시점** → 변호사 검토 (수익 모델이 보일 때)
- **사용자 1,000명 시점** → 정식 개인정보 처리방침 + 변호사 자문료

→ 12월 변호사 검토는 미루는 게 합리적. 출시 자체엔 동의서 + PRIVACY_POLICY_DRAFT v1.0이면 충분.

### 4.4 ⚠ 산출물 버전 표기 비통일

**현황**:
- README: v2.0 → v2.1 (오늘)
- PRD: v5.0 (4.1 후속)
- DESIGN_SYSTEM_SPEC: v1.1
- EPISODE_PRODUCTION_GUIDE: v1.0
- DESIGN_P0_GUIDE: v1.0

→ 7월에 본인이 다시 봤을 때 어느 게 정본인지 혼동 가능.

**권고**: 모든 산출물에 통일된 "5월 정본 v1.x" 라벨 + 변경 이력 표준 양식.
- 이건 시간 투자 대비 가치가 큰 정리 작업이 아니므로, **README v2.1에 "버전 표기 가이드"** 한 섹션 추가하는 정도로 충분.

### 4.5 ⚠ 9-12월 위기 시나리오 가이드 누락

**현황**: README v2.0의 7월 day-by-day는 매우 구체적. 그러나 9-12월 중반의 "예상 못한 위기" 시나리오 가이드가 없음.

**예상 위기**:
- 9월: 첫 코딩 후 환각·오류·타입 에러로 추정보다 시간 소요
- 10월: 에피소드 작성이 지지부진, 베타 리뷰어 응답 지연
- 11월: 동기 저하, "이걸 왜 하고 있지?" 의문
- 12월: 출시 시점 미루기, 변호사 검토 비용 부담

각 시점에 어떻게 대응할지 가이드가 있으면 본인에게 도움됨. → **별도 패치 후보**: `CRISIS_PLAYBOOK.md` 또는 README v2.1에 §추가.

본 점검 결과 즉시 반영하기로 결정한 패치는 1차 4건이고, 4.5는 분량이 커서 별도 결정 필요.
