## 작업 개요

| 항목 | 수치 |
|---|---|
| **출시 시점 에피소드 수** | **5편** (Tier 1, 사안 A로 축소) |
| 출시 후 누적 분량 | 12편 (Tier 2/3 출시 후 매월 1편씩 추가) |

**시간 추정 근거 (v1.1 정정):** 초기 가이드는 3-4시간으로 잡았으나, 이는 본인 작성 시간만 포함. 실제로는 자료 cross-check(60→90-120분) + 베타 리뷰어 응답 대기·수정 반영(편당 30-60분)이 추가됨. EP-04 모범 사례 작성 시간 220분 검증.

**EP-04는 이미 작성됨 (회람판). Tier 1 잔여 4편을 10-11월에 작성하면 됨.**

# Part 1. 에피소드 우선순위 리스트

## Tier 1 — 출시 시점 (Phase 1, 5편)

출시 시점에 무조건 들어가야 할 핵심 에피소드. 시즌 분산 + 임팩트 우선.

| # | 제목 | 도메인 | 시즌 | 핵심 가치 | 제작 난이도 | 상태 |
|---|---|---|---|---|---|---|
| 1 | 회람판·町内会 | 지역공동체 | 연중 | 거주자 모두 마주침. 한국에 없는 개념 | ★★★ | ✅ EP-04 작성됨 |
| 2 | お正月·初詣 | 달력 | 1월 | 출시 직후 첫 시즌 콘텐츠. 가장 유명 | ★★ | 10월 작성 |
| 3 | 보육원 入園式·入学式 | 육아 | 4월 | 자녀 있는 사용자 거의 100% 매칭 | ★★ | 10월 작성 |
| 4 | 飲み会·会食 매너 | 직장 | 연중 | 직장인 100% 매칭. 즉시 활용 가치 | ★★ | 11월 작성 |
| 5 | 在留カード·入管 | 관공서 | 연중 | 외국인 거주자 핵심 제도 | ★★★ | 11월 작성 |

**선정 기준 (5편 압축):**
- ★★ 난이도 우선 (제작 부담 적음): お正月, 入園式, 飲み会
- 연중 활용 가능 우선 (시즌 의존 낮음): 회람판, 飲み会, 在留카드
- 사용자 매칭 폭 우선: 회람판(거주자 100%) > 飲み会(직장인 100%) > 在留카드(외국인 100%) > お正月(전체) > 入園式(자녀 있는 사용자)

**제외 결정 (출시 후로 미룸):**
- ふるさと納税 (★★★★) — 제작 난이도 가장 높음. 11-12월 시즌이라 첫해는 지나쳐도 됨
- お盆 (★★★) — 8월 시즌이라 출시(1월 말) 7개월 후. 시즌 도래 시 추가 가능
- 七五三 (★★) — 11월 시즌. 같은 이유

# Part 2. 에피소드 작성 템플릿

기획서 에피소드 #01을 기준으로 한 표준 양식. 모든 에피소드를 이 구조로 통일.

## 템플릿 구조

```yaml
metadata:
  episode_id: EP-{NN}              # EP-01 ~ EP-15
  title_kr: "한국어 부제"
  title_jp: "日本語タイトル"
  domain: 돈제도 / 지역공동체 / 음식계절 / 목욕문화 / 동요언어 / 의례매너 / 상업쇼핑
  season_tag: 봄 / 여름 / 가을 / 겨울 / 연중
  estimated_minutes: 5-7
  difficulty: phase1-기초 / phase2-실용 / phase3-심화
  target_user: 직장인 / 부모 / 거주자전반
  has_mnemonic: true / false

  # AI 카드 자동 태깅용
  auto_tags:
    categories: [돈제도, 직장]            # 1-2개
    place: 직장 / 관공서 / 보육원 / null
    special: [문화교정?, 니모닉있음?, 외국인가능?]
```

## 5단계 본문 구조

### ① 오늘의 상황 (씬 도입) — 80~150자

**목적:** 사용자가 "아, 이거 나도 겪었어!"라고 즉시 공감하게 만드는 구체 장면.

**작성 규칙:**
- 학습자 1인칭 시점 또는 3인칭 관찰 시점
- 일본어 한 줄(실제 일본인 발화) 포함 필수
- 학습자의 머릿속 생각을 한 문장으로 표현 ("……이게 뭐지?", "왜?", "어떡하지?")
- 길이: 3-4 문장 이내

**좋은 예시 (#01 ふるさと納税):**
> 회사 연말 정산 시즌. 동료가 말한다.
> "今年もふるさと納税やった？お肉もらったよ〜"
> ……'고향 납세'? 왜 고기를 받지? 세금인데? 머릿속이 하얘진다.

**나쁜 예시:**
> ふるさと納税는 일본의 세금 제도 중 하나입니다. 많은 일본인이 이를 활용합니다. (← 교과서식 설명, 공감 X)

---

### ② 오바아찬 스토리 (역사·문화 배경) — 400~600자

**목적:** 제도/문화의 "왜?"를 할머니 말투로 풀어내기. 학습자가 "외우는 게 아니라 이해하게" 만드는 핵심 단계.

**작성 규칙:**
- 도입은 반드시 할머니 말투 인용으로 시작 (`👵 "そういえばね、昔はね〜"` 패턴)
- 역사적 시작점 → 변화 과정 → 현재 의미 순으로 시간 흐름 따라가기
- 한 번 더 할머니 말투 인용을 중간 또는 끝에 배치 (대화 호흡)
- 사실 정보는 별도 단락(▼ 제도의 핵심 구조 같은 sub-heading)으로 분리
- 외국인 활용 정보 (해당하는 경우만): "외국인도 이용할 수 있나?" 단락
- 시각화 가능한 정보는 ASCII 다이어그램 포함

**필수 포함 요소 체크:**
- [ ] 역사적 시작 시점 (몇 년, 누가, 왜)
- [ ] 시간 흐름에 따른 변화 1-2개
- [ ] 현재 일본인의 행동 패턴
- [ ] 외국인 거주자 관련 정보 (해당 시)
- [ ] 할머니 말투 인용 최소 2개

---

### ③ 실전 표현 3개 — 각 30~80자

**목적:** 학습자가 "내일 당장 쓸 수 있는" 표현 3가지. 카드 저장의 핵심 자산.

**작성 규칙:**
- 3개 컨텍스트 고정: 💼 직장 / 🏘 이웃 / 💻 메신저·메일
- 각 표현은 일본어 + 한국어 번역 페어
- 일본어는 자연스러운 구어체, 정중도는 컨텍스트에 맞게
- 카드 저장 시 3개 모두 자동으로 카드 3장 생성됨

**좋은 예시 (#01 ふるさと納税 직장 버전):**
> "ふるさと納税、今年初めてやってみようと思って。 おすすめの自治体とかありますか？"
> *(후루사토 노제, 올해 처음 해보려고요. 추천 지자체 있어요?)*

**필수 체크:**
- [ ] 직장 상사·동료 모두에게 안전한 정중도
- [ ] 이웃·보육원 선생님에게 적절한 친근도
- [ ] 메신저는 직장 채팅(Slack 등) 가정, 너무 격식 아님
- [ ] 3개 표현이 서로 다른 문법 포인트 활용

---

### ④ 깊이 들어가기 — 200~400자

**목적:** 호기심 많은 사용자를 위한 보너스 콘텐츠. "더 알아보기" 영역.

**작성 규칙:**
- 3개 항목 권장 (불릿 또는 sub-heading)
- 비판적 시각, 예외 케이스, 최근 변화 등 깊이 있는 정보
- 학습자 대부분은 안 읽어도 OK (필수 아님)
- 콘텐츠가 부족한 도메인은 ④를 짧게 (200자) 또는 생략

**유형별 좋은 패턴:**
- **제도 에피소드**: 비판 / 최근 변화 / 활용 팁
- **달력 에피소드**: 지역별 차이 / 한국과 비교 / 잘 알려지지 않은 유래
- **육아 에피소드**: 한국 육아와 차이 / 보육원에서의 매너 / 사진관 팁

---

### ⑤ 한국어 니모닉 (선택) — 80~150자

**목적:** 핵심 일본어 단어를 한국 한자어와 연결해서 외울 필요 없게 만들기.

**작성 규칙:**
- 한자가 핵심인 단어에만 적용 (히라가나 단어는 생략)
- 한국 한자와 1:1 매칭이 가능한 경우 우선
- "한자 그대로!" 같은 임팩트 카피 권장
- 한자 분해 형식: `納(납) + 税(세) = 세금을 바치다`

**좋은 예시 (#01):**
> **ふるさと納税** = "**고향(ふるさと)**에 **납세(納税)**" 한자 그대로!
> 納税(납세)는 한국 한자와 동일. 納(납) + 税(세) = 세금을 바치다.
> 고향에 세금을 '기부'하는 제도!

**니모닉 없는 에피소드:** has_mnemonic = false로 설정. 카드 태그에서 #니모닉있음 제외.

# Part 3. 사실 검증 체크리스트

각 에피소드는 발행 전 베타 리뷰어가 검수해야 합니다.

## 베타 리뷰어 섭외 가이드

**필요 인원:** 일본 거주 한국인 1-2명 (자녀 있는 분 포함이면 best)

**섭외 시 안내사항:**
- 검수 분량: 에피소드 1편당 약 15분
- **보상 (v1.1 정정):** 친구/지인 관계라면 **선물 + 식사 한 번**(누적 ¥10,000~¥15,000 상당)이 권장. 금전 거래(편당 ¥5,000)는 친구 관계를 외주 관계로 바꿔 검수 품질에 악영향 가능. 외부 전문가 섭외 시는 편당 ¥5,000~¥8,000.
- 일정: 매주 1-2개씩 검수, 약 8-10주에 걸쳐
- 출시 후 클로즈드 베타 사용자로도 함께 활용

## 검수 체크리스트 (에피소드당)

리뷰어에게 전달할 단일 양식.

```markdown
# 에피소드 검수: EP-{NN} {제목}

## 기본 사실 검증

- [ ] **역사 정보 정확성**: 연도, 인물명, 시작 시점이 맞는가?
- [ ] **현재 정보 최신성**: 2024-2026 기준 현재 사실인가? (제도 변경 잦음)
- [ ] **수치 정확성**: 금액, 비율, 통계 수치 맞는가?
- [ ] **외국인 적용 여부**: 외국인 거주자 관련 정보가 사실인가?
- [ ] **지역 차이 명시**: 도쿄/오사카/지방 차이가 있다면 표시됐는가?

## 일본어 검증

- [ ] **표현 자연스러움**: 실전 표현 3개가 일본인이 실제로 쓰는 말인가?
- [ ] **정중도 적절성**: 직장/이웃/메신저 컨텍스트에 맞는 경어 레벨인가?
- [ ] **할머니 말투 자연스러움**: ② 단계의 할머니 인용이 진짜 할머니가 할 법한 말인가?
- [ ] **어휘 정확성**: 한자 읽기, 외래어 표기 맞는가?
- [ ] **세대 차이**: 젊은 세대와 어른 세대가 다르게 쓰는 표현 구분됐는가?

## 한국 학습자 관점

- [ ] **공감도**: 일본 거주 한국인이 "맞아, 이거 헷갈렸어"라고 느낄 만한가?
- [ ] **한국과의 비교**: 한국 문화와의 차이가 명확히 드러나는가?
- [ ] **즉시 활용도**: 학습자가 내일 당장 쓸 수 있는가?
- [ ] **니모닉 적절성**: 한국 한자 매칭이 자연스러운가? (해당 시)
```

# Part 4. 작성 워크플로우

각 에피소드 1편 = 3-4시간 작업. 다음 단계로 분리.

| 단계 | 작업 | 시간 (수정) |
|---|---|---|
| 1 | 자료 조사 (위키, 정부 자료, 블로그 cross-check) | 90-120분 |
| 2 | ① 씬 도입 작성 | 15분 |
| 3 | ② 스토리 작성 | 60분 |
| 4 | ③ 실전 표현 3개 작성 + 다듬기 | 30분 |
| 5 | ④ 깊이 들어가기 작성 | 30분 |
| 6 | ⑤ 니모닉 (해당 시) | 15분 |
| 7 | 자기 검수 + 메타 데이터 정리 | 20분 |
| 8 | 베타 리뷰어 응답 대기 + 수정 반영 | 30-60분 |
| **합계** | | **약 290-350분 = 5-6시간** |
