# My Niche — 개인정보 처리방침 초안 v1.0

본 문서는 **MVP 출시 전 반드시 정식 변호사 검토를 거쳐야 하는 초안**입니다. 1인 개발 환경에서 production-ready 앱을 운영하려면 최소한 이 정도의 정책 프레임이 필요하다는 가이드라인입니다.

## 적용 법규

| 법규 | 적용 조건 | 핵심 의무 |
|---|---|---|
| 일본 改正個人情報保護法 | 일본 거주 사용자 1명이라도 있으면 적용 | 동의·보존기간 명시·삭제권·국외이전 통지 |
| 한국 PIPA (개인정보보호법) | 한국 거주 사용자 또는 한국어 서비스 | 동의·열람권·삭제권·처리위탁 명시 |
| GDPR (EU) | EU 거주 사용자가 있으면 적용 | 동의·삭제권(Article 17)·이동권·통지 |
| 미국 CCPA | 캘리포니아 사용자 | 판매 거부권·열람권 |

**MVP 단계 권장:** 일본법 + 한국법 + GDPR 3개 만족하는 공통 정책으로 시작. CCPA는 미국 출시 시 추가.

## 제3자 제공·위탁

| 위탁사 | 위탁 항목 | 목적 | 국외이전 |
|---|---|---|---|
| Anthropic (Claude API) | 사용자 입력 한국어 문장 | AI 번역·교정 | 미국 (Anthropic 데이터 정책 적용) |
| Supabase | 모든 DB 데이터 | 호스팅·인증 | 사용자 선택 region (일본/한국 권장) |
| Vercel | 정적 파일 + Edge Function | 배포·라우팅 | 글로벌 CDN |

**Anthropic 처리 정책 명시:** Claude API에 보낸 사용자 입력은 Anthropic의 [데이터 처리 정책](https://www.anthropic.com/legal)에 따라 처리됨. Anthropic은 API 입력을 모델 학습에 사용하지 않음(Zero Data Retention 옵션 가능). 그러나 미국 법률 적용 사실은 사용자에게 명시.

## 사용자 권리

다음 권리는 **앱 내 설정 화면 또는 이메일 요청**으로 행사 가능:

### 1. 열람권 (Access)
- 본인의 모든 데이터를 JSON 형식으로 다운로드
- **구현 방법**: 마이페이지 → "내 데이터 다운로드" 버튼 → JSON export

### 2. 정정권 (Rectification)
- 잘못된 정보 수정 요청
- **구현 방법**: 카드 편집 기능, 프로필 설정 화면

### 3. 삭제권 / 잊혀질 권리 (Erasure, GDPR Article 17)
- 계정 + 모든 카드 + 모든 로그 영구 삭제
- **구현 방법**: 마이페이지 → "계정 삭제" → 확인 절차 → 30일 grace period 후 영구 삭제
- **DB 트리거**: `users` 테이블 DELETE 시 CASCADE로 모든 자식 행 자동 삭제 (이미 마이그레이션에 구현됨)

### 4. 이동권 (Portability, GDPR)
- 본인 데이터를 기계 판독 가능 형식으로 받아 다른 서비스로 이전
- **구현 방법**: 위 1번과 동일한 JSON export

### 5. 처리 거부권 (Objection)
- AI 학습 거부 (애초에 학습 안 함, 명시만)
- 마케팅 거부 (해당 사항 없음, 명시만)

## 데이터 보안

| 영역 | 조치 |
|---|---|
| 전송 | HTTPS 전구간 (Vercel·Supabase 기본 제공) |
| 저장 | Supabase encryption at rest (Postgres TDE) |
| 인증 | Supabase Auth (bcrypt 비밀번호, JWT 세션) |
| 접근 제어 | RLS 정책 (이미 마이그레이션에 18개 정책 구현됨) |
| 백업 | Supabase 자동 일일 백업, 7일 보관 |
| 사고 통지 | 침해 발견 후 72시간 내 사용자 통지 (GDPR 요구) |

## MVP 구현 체크리스트

다음 항목이 **출시 전 모두 구현**되어야 함:

- [ ] 가입 시 본 처리방침 동의 체크박스 (필수)
- [ ] 마이페이지 → "내 데이터 다운로드" 버튼 (JSON export)
- [ ] 마이페이지 → "계정 삭제" 버튼 + 30일 grace period
- [ ] 처리방침 페이지 (앱 내 링크)
- [ ] 비밀번호 재설정 이메일 (Supabase Auth 기본 제공)
- [ ] HTTPS 전구간 (Vercel 자동)
- [ ] RLS 정책 활성화 확인 (이미 마이그레이션에 포함)
- [ ] Anthropic 데이터 처리 정책 링크 명시
- [ ] **변호사 검토** (1인 개발자라도 출시 전 1회 권장)
