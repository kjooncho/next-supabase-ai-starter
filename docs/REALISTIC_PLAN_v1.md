## 본 문서의 사용법

이 문서는 **매주 일요일 밤 또는 월요일 아침**에 펼쳐 보는 메인 가이드입니다.

1. "이번 주 어디까지 왔는지" 확인 (마일스톤 표)
2. **이번 주의 1가지 마이크로 우선순위** 확인
3. 시간 많은 날: 큰 작업 (코딩 집중)
4. 시간 적은 날: 작은 작업 (콘텐츠 자료 조사, 디자인 점검)
5. 위기 시 → `CRISIS_PLAYBOOK.md` 펼치기

매주 1시간 이상 작업 못 한 주가 있으면 **그것을 죄책감으로 만들지 말 것**. 다음 주 피크 패턴으로 보정합니다.

- 90 케이스 Eval 첫 실행 (~$0.16)

**시간 적은 날 (30분-1시간):**
- 도쿄 지인 베타 리뷰어 후보 3명 리스트업 (이름·연락 수단·관계)

**완료 기준:**
- [ ] Eval 결과 v1_results.json 생성됨

- [ ] 베타 리뷰어 후보 3명 결정

**위기 신호**: Eval API 호출이 매번 에러 → `EVAL_README.md` "기준 미달 시 개선 흐름" 또는 `CRISIS_PLAYBOOK §5` (Eval 정확도 저하)

## §4. 육아 변동성 흡수 패턴

### 피크 vs 밸리 패턴

육아 시간이 랜덤이라 주별 가용 시간이 5시간 ~ 18시간으로 변동. 작업 종류를 나눠 흡수합니다.

**피크 주 (가용 12시간 이상) — 큰 작업**
- 새 기능 코딩 (채팅 5단계, My Deck, 선생님 모드)
- Figma 큰 변경 (P0/P1 적용)
- 통합 작업 (엔드 투 엔드 흐름)

**밸리 주 (가용 5시간 이하) — 작은 작업**
- 콘텐츠 자료 조사 (위키 읽기, 일본어 표현 검증)
- 베타 응답 반영 (체크리스트 따라가는 단순 수정)
- 디자인 점검 (시안 비교, 토큰 일관성)
- 문서 작성 (회고, README 업데이트)

**중간 주 (가용 6-11시간) — 혼합**
- 작은 컴포넌트 코딩 (헤더, 탭바, 카드)
- EP 1편 1차 작성 또는 마감

### 시간 단위 작업 분해 ("다음 5시간" 단위)

주 단위 계획이 아닌, **본인이 시간 났을 때 펼치는 마이크로 작업** 리스트:

**5분 단위:**
- 베타 리뷰어 답신 확인
- README 변경 이력 업데이트
- Figma 컬러 토큰 1개 확인

**30분 단위:**
- preflight_check 실행
- 자료 조사 1개 위키 읽기
- 컴포넌트 1개 prop 정의

**2시간 단위:**
- 컴포넌트 1개 완성 (헤더, 모달 등)
- EP 1편 ① 씬 도입 + ② 오바아찬 스토리 작성
- Figma P0 1건 적용

**5시간 단위:**
- EP 1편 1차 작성 완료
- 5단계 번역 중 1단계 구현
- My Deck 통계 카드 + 그리드 동시 구현

**8시간 단위 (주말 1일):**
- 채팅 탭 전체 흐름 완성
- 베타 5명 모두 피드백 반영
- 큰 리팩터링

### "오늘 시점에 가장 중요한 1가지" 결정 흐름

매일 작업 시작 시 30초 결정:

1. **이번 주 마이크로 우선순위가 무엇이지?** (위 §3 참조)
2. **지금 가용 시간은?** (5분 / 30분 / 2시간 / 5시간 / 8시간)
3. **그 시간 안에 끝낼 1가지가 무엇인가?**
4. **그것 시작.**

오늘 못 끝낸 게 있으면 죄책감 NO. 다음 피크 주에 흡수.

## §7. 본 플랜의 한계와 가정

투명하게 명시합니다:

### 가정 1: 주 평균 11시간 가용
육아 변동성으로 일부 주는 5시간, 일부 주는 18시간. 평균 11시간이 유지되면 마일스톤 가능. 평균 8시간 이하로 떨어지면 출시 12월로 미루는 결정 필요 (`CRISIS_PLAYBOOK §6`).

### 가정 2: 도쿄 지인 베타 응답 5-7일 이내
지인이라 일반 커뮤니티 응답률(30%)보다 높을 것 (예: 70%). 그러나 정확히 5-7일은 보장 안 됨. 응답 지연 시 마일스톤 1주 후행.

### 가정 3: Eval Pass Criteria v1 통과 (type ≥70%)
첫 실행에서 통과 가정. 미달 시 v2 작성 1-2주 추가 → 6월 1주 시작 → 11월 1주 출시 가능 (1주 보정).

### 가정 4: AI 페어 코딩 효과적 활용
Claude Code, Cursor 등으로 코딩 시간 단축. 그렇지 않으면 시간 1.3-1.5배.

### 가정 5: 캘린더/생활탭/Sentry 미루기 결정 유지
출시 시점에 이걸 다시 넣고 싶어지면 출시 12월로 미뤄야 함. v1.1에서 추가하는 게 부담 적음.

### 무엇이 잘못될 수 있는가
- 가장 큰 위험: 7-8월 채팅 탭 코딩이 추정보다 더 오래 걸림
